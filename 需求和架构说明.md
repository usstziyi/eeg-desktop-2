# OpenBCI Cyton 实时 EEG 桌面应用技术框架说明

本文档基于 OpenBCI、BrainFlow、PyQtGraph 官方文档及对应 GitHub 仓库重新整理，目标是为一个面向 OpenBCI Cyton 8 通道 EEG 设备的实时桌面应用提供可落地的技术框架。

---

## 1. 项目目标

构建一个 Python 桌面应用，用于连接 OpenBCI Cyton 8 通道设备，完成以下能力：

- 实时采集 EEG 数据。
- 实时显示 8 通道波形。
- 实时执行基础信号处理：去趋势、带通、工频陷波、频谱分析、频带能量计算。
- 支持合成数据模式，方便无硬件时开发与演示。
- 支持数据录制、标记、配置持久化和后续扩展。
- 退出时可靠停止数据流并释放 BrainFlow session。

## 2. 官方资料核对结论

### 2.1 OpenBCI Cyton

OpenBCI 官方文档确认：

- Cyton Board 是 8 通道生物电放大器，可用于 EEG、EMG、ECG。
- 默认采样率为 250 Hz。
- Cyton + Daisy 为 16 通道，采样率通常为 125 Hz。
- Cyton 通过 OpenBCI USB Dongle 与电脑通信，底层表现为串口连接。
- Cyton 数据包包含采样序号、8 路 ADS1299 通道数据、加速度或 AUX 数据等。

对本项目的架构影响：

- 首版固定面向 `CYTON_BOARD`，即 8 通道、250 Hz。
- 串口是必须配置项，例如 Windows 下 `COM3`，Linux 下 `/dev/ttyUSB0`，macOS 下通常使用 `/dev/cu.*`。
- 真实硬件调试前必须提供串口选择、连接状态、异常提示。
- 需要保留未来扩展到 Cyton Daisy、WiFi Shield、Marker 模式、阻抗检测的接口。

### 2.2 BrainFlow

BrainFlow 官方文档确认：

- BrainFlow 提供板卡无关的数据采集 API。
- `BoardShim` 负责设备连接、启动流、停止流、读取数据、释放 session。
- `DataFilter` 提供信号处理 API，包括滤波、PSD、频带能量等。
- 返回数据是二维数组，行是不同数据通道，列是时间采样点。
- 应使用 `BoardShim.get_eeg_channels(board_id)`、`get_accel_channels()`、`get_timestamp_channel()` 等 API 获取通道索引，不应硬编码行号。
- `get_current_board_data(num_samples)` 获取最近数据且不清空 BrainFlow 内部 ring buffer。
- `get_board_data()` 获取并清空 BrainFlow 内部 ring buffer。
- `start_stream(buffer_size, streamer_params)` 会启动 BrainFlow 内部采集线程并将数据写入内部 ring buffer。
- `stop_stream()` 只停止流，不释放资源；`release_session()` 才释放 session。

对本项目的架构影响：

- 采集层以 BrainFlow 内部 ring buffer 为主，应用层再维护一个 UI/处理用的短窗口缓存。
- 实时绘图优先使用 `get_current_board_data(window_samples)`，避免频繁清空内部缓存。
- 数据录制可使用 BrainFlow streamer，例如 `file://path:w`，也可以应用层自行写 CSV/EDF。
- 信号处理既可调用 BrainFlow `DataFilter`，也可在需要流式状态滤波时使用 SciPy。

### 2.3 PyQtGraph

PyQtGraph 官方文档和 GitHub 仓库确认：

- PyQtGraph 是面向科学和工程应用的 Qt 绘图库，依赖 NumPy 和 Qt 绑定。
- 支持 PyQt5、PyQt6、PySide6。
- 官方示例中 BrainFlow 实时绘图采用 `QTimer` 周期读取数据并更新曲线。
- `PlotDataItem` 是常规 2D 曲线的主接口。
- 性能优化建议包括：线宽使用 1 px、关闭不必要的抗锯齿、对高密度数据使用 downsampling、clip-to-view、避免每帧重建图形对象。

对本项目的架构影响：

- GUI 主线程只做状态展示和曲线刷新，不做阻塞 I/O 和重计算。
- 曲线对象启动时创建，刷新时只调用 `setData()`。
- 刷新间隔建议 30-100 ms，首版采用 50 ms。
- 每通道显示 4-10 秒窗口，避免无限增长曲线数据。

## 4. 总体架构

采用“采集层 + 处理层 + GUI 层”的分层结构。

```text
OpenBCI Cyton / Synthetic Board
        |
        v
BrainFlow BoardShim
        |
        v
Acquisition Service
        |
        +--> Raw Sample Cache / Window Buffer
        |
        v
Processing Service
        |
        +--> Filtered EEG
        +--> FFT / PSD
        +--> Band Power
        |
        v
Qt Signal / Slot
        |
        v
PySide6 + PyQtGraph GUI
```

核心原则：

- BrainFlow 负责硬件通信和内部采集缓存。
- GUI 主线程只响应用户操作和刷新可视化。
- 处理逻辑与 GUI 解耦，方便测试。
- 任何地方都不硬编码 Cyton 的 EEG 行号，统一通过 BrainFlow API 查询。
- 所有启动、停止、异常路径都必须执行资源清理。

## 5. 线程模型

推荐首版使用两个工作线程加一个 GUI 主线程。

```text
GUI 主线程
  - MainWindow
  - ControlPanel
  - PyQtGraph 曲线刷新
  - QTimer 50 ms
  - 接收 Qt signals 更新状态

AcquisitionWorker / QThread
  - prepare_session()
  - start_stream()
  - 周期读取 get_current_board_data()
  - 发出 raw_data_ready
  - stop_stream()
  - release_session()

ProcessingWorker / QThread
  - 接收最近窗口数据
  - 去趋势 / 滤波 / PSD / 频带能量
  - 发出 processed_data_ready
```

如果首版希望降低复杂度，也可以采用：

- BrainFlow 自己负责后台采集。
- GUI 用 `QTimer` 调用 `get_current_board_data()`。
- 较重的 PSD/频带计算放入 `ProcessingWorker`。

这种方式更接近 BrainFlow 官方实时绘图示例，适合 MVP。

## 6. 数据流设计

### 6.1 BrainFlow 数据读取

连接 Cyton：

```python
from brainflow.board_shim import BoardShim, BrainFlowInputParams, BoardIds

params = BrainFlowInputParams()
params.serial_port = "COM3"

board = BoardShim(BoardIds.CYTON_BOARD.value, params)
board.prepare_session()
board.start_stream(45000)
```

实时读取：

```python
board_id = board.get_board_id()
sampling_rate = BoardShim.get_sampling_rate(board_id)
eeg_channels = BoardShim.get_eeg_channels(board_id)

window_seconds = 4
num_points = sampling_rate * window_seconds
data = board.get_current_board_data(num_points)
eeg_data = data[eeg_channels, :]
```

停止和释放：

```python
try:
    board.stop_stream()
finally:
    board.release_session()
```

### 6.2 缓存策略

BrainFlow 已有内部 ring buffer，应用层不需要复制一个无限缓存。建议使用三类窗口：

- `display_window`: 4-10 秒，用于波形显示。
- `analysis_window`: 2-8 秒，用于 FFT/PSD 和频带能量。
- `recording_stream`: 写文件时使用 BrainFlow streamer 或独立写入器。

### 6.3 数据单位

BrainFlow 对不同板卡返回的单位由官方数据格式定义。实现中不应假设原始串口字节格式，而应以 BrainFlow 返回数据为准。显示层统一标注为微伏级 EEG 信号，并在需要时通过配置调整缩放。

## 7. 信号处理设计

### 7.1 首版处理链

对每个 EEG 通道：

1. 去趋势：移除 DC 偏置。
2. 带通：例如 1-45 Hz 或 3-45 Hz。
3. 工频陷波：50 Hz 或 60 Hz，按配置选择。
4. PSD：Welch 方法。
5. 频带能量：
   - Delta: 0.5-4 Hz
   - Theta: 4-8 Hz
   - Alpha: 8-13 Hz
   - Beta: 13-30 Hz
   - Gamma: 30-45 Hz

### 7.2 BrainFlow DataFilter

BrainFlow `DataFilter` 支持：

- `detrend`
- `perform_bandpass`
- `perform_bandstop`
- `remove_environmental_noise`
- `get_psd`
- `get_psd_welch`
- `get_band_power`
- `get_avg_band_powers`

注意：

- BrainFlow 的常规滤波 API 是对传入数组原地处理。
- 对“最近 N 秒窗口”做显示级滤波可以使用 `DataFilter`。
- 对严格流式、低延迟、需保留滤波器状态的处理链，建议使用 SciPy `sosfilt` 并保存 `zi/zf` 状态。

### 7.3 实时滤波取舍

推荐两级策略：

- MVP：直接对 `get_current_board_data()` 返回窗口使用 BrainFlow `DataFilter`，实现简单，足够做实时观察。
- 进阶：在 `ProcessingWorker` 中使用 SciPy `butter(..., output="sos")` + `sosfilt(..., zi=state)`，避免每个窗口重复从头滤波导致边缘效应和相位不一致。

不建议实时流中使用 `filtfilt` 作为核心处理，因为它需要未来数据，适合离线零相位处理，不适合因果实时处理。

## 8. GUI 设计

### 8.1 主界面区域

建议布局：

- 顶部工具栏：
  - 设备模式：Synthetic / Cyton
  - 串口选择
  - Connect / Disconnect
  - Start / Stop
  - Record
  - Marker
- 左侧控制面板：
  - 显示窗口秒数
  - 增益 / 缩放
  - 带通范围
  - 工频陷波 50/60 Hz
  - PSD 窗口参数
- 中央波形区：
  - 8 通道 EEG 实时波形
- 右侧分析区：
  - PSD 曲线
  - 频带能量柱状图
  - 连接状态、采样率、丢包/缓存状态

### 8.2 PyQtGraph 实现原则

- 初始化时创建 8 条 `PlotDataItem`。
- 刷新时只调用 `curve.setData(x, y)`。
- 使用 1 px 线宽。
- 关闭不必要的 symbol、legend、网格。
- 大窗口显示时启用 downsampling 或 clip-to-view。
- GUI 刷新频率与采样率解耦，建议 50 ms 一帧。

## 11. 错误处理与资源释放

必须覆盖以下场景：

- 串口不存在。
- 串口被占用。
- 设备连接后无数据。
- `prepare_session()` 成功但 `start_stream()` 失败。
- 用户点击 Stop。
- 用户关闭窗口。
- 处理线程异常。

资源释放顺序：

```text
用户 Stop / 关闭窗口
  -> 停止 GUI QTimer
  -> 停止 ProcessingWorker
  -> board.stop_stream()
  -> board.release_session()
  -> 等待 QThread 退出
  -> 写入配置和日志
```

`release_session()` 必须执行，否则设备或串口可能在下次连接时处于不可用状态。

## 13. 关键实现约束

- 不硬编码 EEG 通道索引，必须使用 `BoardShim.get_eeg_channels(board_id)`。
- GUI 主线程不执行阻塞采集和重型 PSD 计算。
- 曲线对象不在每帧重建。
- 所有 BrainFlow session 必须成对 `prepare_session()` / `release_session()`。
- Stop 与 closeEvent 走同一套清理逻辑。
- 默认先支持 Synthetic Board，保证无硬件也能开发。
- 实时处理默认窗口长度至少 4 秒，保证频谱和频带能量稳定。
- Cyton 默认按 250 Hz 设计，但采样率必须从 BrainFlow API 获取。
